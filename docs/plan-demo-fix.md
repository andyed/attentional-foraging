## Problem (original)

The live demo at andyed.github.io/attentional-foraging had three issues:
1. **Gaze fixations don't align with the Scrutinizer foveated render** — the gazeplot (from tiled capture) is a different height than the SERP render (from Playwright), so SVG overlay coordinates land in the wrong place when toggling between views
2. **Mouse cursor doesn't show** during playback (the data loads but coordinate spaces may mismatch)
3. **The foveated render was generated by tiling** — scrolling through viewport-sized captures and stitching, which introduces height mismatches and tile-boundary artifacts

## Root cause

Two rendering pipelines produce images at different heights:
- **Playwright SERP screenshot**: 1280×docHeight (e.g., 1280×2359 for p037-b2-t5) — correct
- **Scrutinizer gazeplot**: 1280×(tileCount × vpH) — inflated because tiles are viewport-height chunks and `tileCount = ceil(docH/vpH) + 1`

The fixation data in the HTML viewer is **raw FPOGX/FPOGY** (screen-space), not scroll-corrected page-space. The build script (`scripts/build-gh-pages.js` line 97) reads fixations straight from the CSV without applying scroll offset. But the gazeplot images ARE in page-space.

## Solution: Single full-height Scrutinizer render

**De-risked.** Tested on 2026-04-02: Scrutinizer successfully rendered at 1280×2359 (full document height) without crashing or WebGL failure. The overlay reported frame at 2560×3188 (2× DPR). Max curated trial height is 3756px (p029-b2-t10) — should be fine.

### Step 1: Modify `capture-fullpage-gazeplot.js` for single-capture mode

Add a `--single` flag that:
- Sets `TEST_HEIGHT` to the full document height (from trial metadata)
- Skips the tiling loop entirely
- Captures one screenshot after walking all fixations with infinite visual memory
- Outputs a single PNG at 1280×docHeight (or 2560×2×docHeight at 2× DPR — resize down)

The existing script already:
- Loads trial data via `adserp-importer.js` (scroll-corrected page-space fixations)
- Walks fixations through the pipeline with `TEST_VISUAL_MEMORY=-1`
- Captures screenshots via `capturePage()`

The change is: don't tile. Set the window to full page height, walk fixations, capture once.

**Concern:** Electron window height at 3756px may exceed macOS window size limits. If so, use `webContents.capturePage({ x: 0, y: 0, width: 1280, height: docH })` with the content scrolled, rather than resizing the window. Test on the tallest trial (p029-b2-t10, 3756px) first.

### Step 2: Fix coordinate spaces in `build-gh-pages.js`

Three fixes needed:

**a) Fixation coordinates → page-space.** Currently raw FPOGX/FPOGY. Need scroll correction:
```js
// For each fixation, interpolate scroll offset at fixation time
// Same step-function interpolation as the notebook analysis
const scrollAtTime = (t) => {
    const scrolls = mouseEvents.filter(m => m.e === 'scroll');
    // ... bisect to find last scroll before t, return its ypos
};
fixations = fixations.map(f => ({
    ...f,
    y: Math.min(f.y, screenH) + scrollAtTime(f.t),  // CLAMP + scroll offset
    x: f.x * rx,  // scale from window-space to screenshot-space
}));
```

Also apply the **FPOGY clamp** (24.5% of samples exceed screen height — discovered in this session).

**b) Mouse coordinates → page-space.** Currently `m.y` is raw ypos. For mousemove events, this is screen-space. For scroll events, ypos is the scroll offset itself. Need:
```js
mouseEvents = mouseEvents.map(m => {
    if (m.e === 'scroll') return { ...m, y: 0 }; // scroll events aren't cursor positions
    return { ...m, y: Math.min(m.y, screenH) + scrollAtTime(m.t) };
});
```

**c) SVG and image heights must match.** Both the gazeplot and SERP render should be exactly `docHeight` pixels tall (at 1280px wide). The SVG viewBox should be `0 0 1280 docHeight`. No more separate GAZEPLOT_H and SERP_H.
