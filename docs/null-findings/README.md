# Null findings

**Principle (2026-04-15, Andy Edmonds).** This project documents null and near-null findings in the git repo even when they do not appear in papers. Published papers have page limits and rhetorical constraints that make it impractical to report every experiment that didn't work, but the analyses are still on disk and future work — including our own — benefits from a clean record of what was tried, what the numbers actually were, and why the result did not survive to publication. This directory is the canonical home for that record.

## What belongs here

- Experiments whose headline result was a null or near-null that we chose not to surface in a paper
- Experiments whose framing changed mid-analysis, where the original framing is worth preserving for honesty and reproducibility
- Ablations and diagnostic checks that returned the expected-but-unpublishable answer ("this confound is real but doesn't change the published finding")
- Any experiment where an *earlier* summary we made was later revised by a more honest metric; the earlier summary gets a writeup here showing *how* it was revised

## Format conventions

Each null finding gets its own markdown file in this directory, named `nb<##>-<short-slug>.md` or `<date>-<slug>.md` depending on whether it came from a Tier-A notebook or a one-off script. Each file includes:

1. **TL;DR header** — one paragraph. What was the hypothesis, what was the headline number, what did it not demonstrate.
2. **What was run** — link to the notebook or script, the exact CV protocol, sample sizes, feature set, label scheme.
3. **Numbers** — the full results table including the honest null alongside any near-positive results that tempted us to claim more.
4. **Why it's a null** — what changed (noise floor, denominator, sample composition) that killed the positive framing.
5. **What was learned anyway** — every null tells you something. Document the narrow claim that *does* survive.
6. **Pointers** — files touched, commits, upstream discussions (Slack, email), related notebooks.

## Relationship to other project docs

- `docs/findings.md` is where **published** findings live, tagged with `[NB##:K##]` Key Claim IDs.
- `docs/null-findings/` is where **unpublished** findings live. It's the shadow index to findings.md.
- `docs/methodological-threats.md` catalogues known confounds across all notebooks; some entries here will cross-reference it.

## Why this matters

The file-drawer problem in empirical research is the accumulated under-reporting of null results across many labs and papers. On a single-lab project like this one, the shadow cost of not documenting nulls is re-walking the same paths multiple times over the project's life and forgetting what was tried. Five minutes writing a null up cleanly today saves hours of re-derivation six months from now, and — more importantly — gives any reader of our published work a citable place to see the full empirical envelope of the project, not just the results that landed in print.

- [`2026-04-16-satopt-lhipa-duration-confound.md`](2026-04-16-satopt-lhipa-duration-confound.md) — 2026-04-16 — NB11:K12's per-participant ρ(regression_rate, mean_LHIPA) = −0.568 reproduces (we measured −0.593, n = 47), but the partial correlation controlling for trial duration collapses to ρ = +0.135 (p = 0.37) while ρ(duration, mean_LHIPA \| regression_rate) stays at −0.862 (p = 1.4 × 10⁻¹⁴). - [`nb29-viewport-bands-content-residualization.md`](nb29-viewport-bands-content-residualization.md) — 2026-04-19 — NB29 tested whether residualizing viewport-band dwell (the NB28 signal) against per-result content features (token_count, char_count, TTR, query_cosine, centroid_novelty) would improve deferred-vs-evaluated-rejected discrimination beyond raw bands. Clean null: Δ (retreat + residualized) − (retreat + raw) = **−0.024 AUC**, bands-alone Δ = −0.103 AUC, per-participant median Δ = −0.066, only 21 % of participants benefit. Content explains ≤ 2.5 % of band-dwell variance; rank + participant FE dominate. Methodological caveat: the baseline's participant FE absorbs the variance where the signal lives, so the null is partly tautological under this protocol. What did survive: the viewport-band signal is content-invariant, which is *desirable* for cross-corpus generalization.

- [`2026-04-19-lfhf-content-crossover-null.md`](2026-04-19-lfhf-content-crossover-null.md) — 2026-04-19 — ETTAC gap audit. The NB25 / NB27 / NB29 content/embedding work had never been crossed with NB14's Butterworth LF/HF signal; that crossover is now run. Inner-joining 6,056 `(trial, P0–P9)` records against per-result content features (token_count, char_count, TTR, query_cosine, centroid_novelty) and testing each feature three ways (pooled Spearman, partial Spearman controlling for position, fixed-effects OLS with participant + position dummies) — **no feature × test survives Bonferroni α = 0.003 across 15 tests**. Largest unadjusted signal is FE β on centroid_novelty (p = 0.018) which fails correction. Per-position |ρ| rarely exceeds 0.10 and sign-flips across positions — noise around zero. Positive control on the same joined subset reproduces K3 exactly (ρ = −0.927, p < 10⁻⁴). Interpretation: LF/HF is position-bound (framework compilation sharpens by rank), not item-semantics-bound. Consistent with the NB29 and 2026-04-15 novelty-residual-redundancy nulls.
- [`2026-04-19-lfhf-satopt-orthogonality-p03-robust.md`](2026-04-19-lfhf-satopt-orthogonality-p03-robust.md) — 2026-04-19 — ETTAC queue item 3. The LF/HF × satopt orthogonality result (originally AUC = 0.43, χ² *p* = 0.77, Spearman(slope, rate) ρ = −0.226 *p* = 0.13) re-run on the steep-phase (P0–P3) subset. Steep-phase orthogonality is *stronger* than the original full-range: LOO-LR AUC = 0.523 (majority baseline 0.522, at chance), Spearman(slope, rate) ρ = −0.020 (*p* = 0.90), every per-feature |*d*| < 0.21. Load-trajectory axis is genuinely orthogonal to satopt / regression-rate segmentation on the range where K10 lives.
- [`2026-04-20-rung4-rank-within-trial.md`](2026-04-20-rung4-rank-within-trial.md) — 2026-04-20 — Phase 3 of the `withstood_evaluation` work. Hypothesis: denser 10-grade LambdaMART labels derived from rank-within-trial `withstood_pre_click` will beat the NB26 3-grade baseline on the text + M4 + VP feature set by exposing ~45 informative pairs per trial instead of ≤3. Falsified at three variants (R4a exp gain, R4b on `withstood_full`, R4e linear gain), all Δ MRR ≈ −0.027 (p ≈ 0.98). Ridge MSE pointwise matched 3-grade (+0.008 ns). The remedy (hybrid label, click pinned at 9) restores the denser-label win at +0.143 MRR — documented as the positive companion finding.
